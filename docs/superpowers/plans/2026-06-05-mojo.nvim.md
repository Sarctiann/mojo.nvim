# mojo.nvim Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a generic Neovim plugin that centralizes Mojo support and can later swap custom modules for official tools.

**Architecture:** Small Lua modules with clear boundaries. Core setup handles filetype, environment helpers, Treesitter registration, and terminal activation. LSP/formatter integrations are exposed as adapters so LazyVim or any other manager can opt in without hard dependency.

**Tech Stack:** Lua, Neovim API, optional `nvim-treesitter`, optional `nvim-lspconfig`, optional `conform.nvim`, GitHub public repo via `gh`.
